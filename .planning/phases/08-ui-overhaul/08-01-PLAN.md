<task type="auto">
  <name>Task 1: Configure distinctive typography with Switzer + JetBrains Mono</name>
  <files>app/layout.tsx, app/globals.css</files>
  <action>
    Replace Geist fonts with:
    - **Switzer** for headings and body (80s/90s magazine-inspired, memorable aesthetic)
    - **JetBrains Mono** for numbers and data (highly readable for financial figures)

    In app/layout.tsx:
    1. Import Switzer and JetBrains_Mono from next/font/google
    2. Configure with subsets: ['latin'], display: 'swap'
    3. Set variable names: --font-switzer and --font-jetbrains-mono
    4. Apply both variables to html element className

    In app/globals.css @theme inline block:
    1. Update --font-sans to var(--font-switzer)
    2. Update --font-mono to var(--font-jetbrains-mono)
    3. Keep body font-family using var(--font-sans)

    **Avoid using:**
    - Inter, Roboto, Arial, Space Grotesk (generic AI fonts)
    - display: 'optional' (can skip font load - use 'swap' for brand fonts)
    - Manual Google Fonts links in head (Next.js handles optimization)
  </action>
  <verify>npm run build succeeds, inspect network tab shows Switzer + JetBrains Mono woff2 files loaded, check Elements panel shows --font-switzer and --font-jetbrains-mono CSS variables applied</verify>
  <done>Both fonts load successfully, CSS variables reference correct fonts, body text uses Switzer, no build errors</done>

    - --muted: oklch(92% 0.01 270) (light gray with purple tint)
    - --muted-foreground: oklch(40% 0.01 270)
    - --border: oklch(88% 0.01 270)
    - --input: oklch(88% 0.01 270)
    - --ring: oklch(45% 0.18 270)
    - --destructive: oklch(60% 0.20 25) (red for delete actions)
    - --destructive-foreground: oklch(98% 0.01 270)

    In @media (prefers-color-scheme: dark) - TOKYO NIGHT PALETTE:
    - --background: oklch(15% 0.02 270) (deep purple-black)
    - --foreground: oklch(95% 0.01 270) (off-white)
    - --primary: oklch(60% 0.18 270) (brighter purple for dark mode)
    - --primary-foreground: oklch(15% 0.02 270)
    - --accent: oklch(75% 0.15 195) (bright cyan)
    - --accent-foreground: oklch(15% 0.02 270)
    - --muted: oklch(20% 0.02 270) (dark card background)
    - --muted-foreground: oklch(70% 0.01 270)
    - --border: oklch(25% 0.03 270)
    - --input: oklch(25% 0.03 270)
    - --ring: oklch(60% 0.18 270)
    - --destructive: oklch(65% 0.20 25)
    - --destructive-foreground: oklch(15% 0.02 270)

    In @theme inline block, add all color mappings:
    - --color-background: var(--background)
    - --color-foreground: var(--foreground)
    - --color-primary: var(--primary)
    - --color-primary-foreground: var(--primary-foreground)
    - --color-accent: var(--accent)

    **Why OKLCH over HSL:**
    - Perceptually uniform (all colors look equally bright)
    - Wider P3 gamut for vivid colors
    - Better gradient blending (no gray areas)
    - Tailwind 4 native format

    **Avoid:**
    - HSL format (Tailwind 4 uses OKLCH natively)
    - Hardcoded hex colors (always use CSS variables)
    - Chroma > 0.25 in OKLCH (vibrates on dark backgrounds)
